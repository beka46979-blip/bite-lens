# Обновление функционала сброса пароля

## Что изменено

### ✅ Валидация сложного пароля

Теперь при сбросе пароля требуется **сложный пароль** со следующими требованиями:

1. **Минимум 8 символов**
2. **Хотя бы одна строчная буква** (a-z)
3. **Хотя бы одна заглавная буква** (A-Z)
4. **Хотя бы одна цифра** (0-9)
5. **Хотя бы один специальный символ** (!@#$%^&* и т.д.)

### ✅ Визуальный индикатор требований

При вводе нового пароля отображается интерактивный список требований с галочками:
- ✅ Зеленая галочка - требование выполнено
- ❌ Серая галочка - требование не выполнено

### ✅ Автоматический вход после сброса

После успешного сброса пароля:
1. Создается новая сессия
2. Генерируется JWT токен
3. Токен сохраняется в cookie
4. Пользователь автоматически перенаправляется в **профиль** (`/dashboard`)
5. Все старые сессии удаляются для безопасности

### ✅ Централизованная валидация

Создана утилита `lib/auth/password-validation.ts` для:
- Валидации пароля на клиенте и сервере
- Проверки требований к паролю
- Использования в регистрации и сбросе пароля

## Измененные файлы

### Frontend
- `app/forgot-password/ForgotPasswordForm.tsx` - добавлена валидация и визуальный индикатор

### Backend

### Утилиты
- `lib/auth/password-validation.ts` - **новый файл** с функциями валидации

## Как это работает

### 1. Пользователь запрашивает сброс пароля
```
/forgot-password → вводит email → получает код
```

### 2. Вводит код и новый пароль
```
Вводит 6-значный код → вводит новый пароль
```

### 3. Валидация пароля
```typescript
// Клиент (React)
const passwordRequirements = getPasswordRequirements(newPassword);

// Сервер (API)
const validation = validateStrongPassword(newPassword);
if (!validation.valid) {
  return error(validation.error);
}
```

### 4. Автоматический вход
```typescript
// Создание токена
const token = await signJWT({ userId, email, role });

// Сохранение в cookie
cookieStore.set('auth_token', token, { httpOnly: true });

// Перенаправление
router.push('/dashboard');
```

## Безопасность

✅ Все старые сессии удаляются при сбросе пароля  
✅ Пароль хешируется с помощью bcrypt  
✅ JWT токен сохраняется в httpOnly cookie  
✅ Валидация на клиенте и сервере  
✅ Защита от простых паролей  

## Тестирование

1. Откройте `/forgot-password`
2. Введите email зарегистрированного пользователя
3. Получите код (в email или в демо-режиме на экране)
4. Введите код
5. Введите новый пароль (попробуйте простой - увидите ошибки)
6. Введите сложный пароль согласно требованиям
7. После успешного сброса вы будете автоматически перенаправлены в `/dashboard`

## Примеры паролей

❌ **Слабые пароли** (не пройдут валидацию):
- `12345678` - нет букв и спецсимволов
- `password` - нет цифр, заглавных букв и спецсимволов
- `Password` - нет цифр и спецсимволов
- `Password1` - нет спецсимволов

✅ **Сильные пароли** (пройдут валидацию):
- `MyPass123!`
- `Secure@2024`
- `Test#Password1`
- `Admin$Pass99`

- Изменении пароля в профиле (если реализовано)
- Создании админа через скрипты
