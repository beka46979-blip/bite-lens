## 1. Убедитесь что БД запущена

```bash
# Проверьте что PostgreSQL запущен
# DATABASE_URL в .env должен быть правильным
```

## 2. Примените миграции (если еще не применены)

```bash
npx prisma migrate dev
```

## 3. Запустите dev сервер

```bash
npm run dev
```

## 4. Тестирование

### Регистрация нового пользователя
1. Откройте http://localhost:3000
2. Нажмите "Начать" или "Зарегистрироваться"
3. Заполните форму:
   - Email: test@example.com
   - Пароль: 12345678 (минимум 8 символов)
   - Подтвердите пароль
4. Нажмите "Зарегистрироваться"
5. Вы будете перенаправлены на `/dashboard`

### Вход существующего пользователя
1. Откройте http://localhost:3000/login
2. Введите email и пароль
3. Нажмите "Войти"
4. Вы будете перенаправлены на `/dashboard`

### Смена языка
1. В header нажмите на иконку глобуса
2. Выберите язык (Русский / Кыргызча)
3. Интерфейс изменится мгновенно
4. Язык сохранится в cookie

### Выход
1. На странице dashboard нажмите "Выйти"
2. Вы будете перенаправлены на `/login`

## Доступные страницы

- `/` - Главная (публичная)
- `/login` - Вход (публичная)
- `/register` - Регистрация (публичная)
- `/dashboard` - Панель управления (защищенная)

## Защита маршрутов

Middleware автоматически:
- ✅ Блокирует доступ к `/dashboard` без авторизации
- ✅ Перенаправляет авторизованных с `/login` на `/dashboard`
- ✅ Сохраняет токены в httpOnly cookies

## Troubleshooting

### Ошибка "Unauthorized"
- Проверьте что JWT_SECRET установлен в .env
- Очистите cookies браузера
- Перезапустите dev сервер

### Ошибка подключения к БД
- Проверьте DATABASE_URL в .env
- Убедитесь что PostgreSQL запущен
- Проверьте что БД создана

### Ошибка "Email already exists"
- Пользователь с таким email уже зарегистрирован
- Используйте другой email или войдите
