# Индексы базы данных Bite Lens

## Добавленные индексы для оптимизации производительности

### 1. **ai_logs**
- `@@index([user_id])` - для быстрого поиска логов по пользователю
- `@@index([created_at])` - для сортировки и фильтрации по дате

### 2. **daily_nutrition_summary**
- `@@index([user_id])` - для быстрого получения сводок пользователя
- `@@index([date])` - для фильтрации по датам

### 3. **food_snap_feedback**
- `@@index([user_id])` - для получения отзывов пользователя
- `@@index([snap_id])` - для получения отзывов на конкретный снимок

### 4. **food_snaps**
- `@@index([user_id])` - для получения снимков пользователя
- `@@index([created_at])` - для сортировки по дате
- `@@index([meal_id])` - для получения снимков конкретного приема пищи

### 5. **meals**
- `@@index([user_id])` - для получения приемов пищи пользователя
- `@@index([created_at])` - для сортировки по дате

### 6. **notifications**
- `@@index([user_id])` - для получения уведомлений пользователя
- `@@index([created_at])` - для сортировки по дате
- `@@index([is_read])` - для фильтрации прочитанных/непрочитанных

### 7. **sessions**
- `@@index([user_id])` - для получения сессий пользователя
- `@@index([expires_at])` - для очистки истекших сессий

### 8. **subscriptions**
- `@@index([user_id])` - для получения подписок пользователя
- `@@index([status])` - для фильтрации по статусу подписки

### 9. **user_activity_logs**
- `@@index([user_id])` - для получения активности пользователя
- `@@index([created_at])` - для сортировки по дате

### 10. **user_goals**
- `@@index([user_id])` - для получения целей пользователя

### 11. **users**
- `@@index([email])` - для быстрого поиска по email (дополнительно к unique)
- `@@index([created_at])` - для сортировки пользователей по дате регистрации
- `@@index([role])` - для фильтрации по ролям (USER/ADMIN)

### 12. **weight_logs**
- `@@index([user_id])` - для получения записей веса пользователя
- `@@index([log_date])` - для сортировки и фильтрации по дате

## Преимущества

### Производительность
- ✅ Ускорение запросов с фильтрацией по `user_id` (самый частый случай)
- ✅ Быстрая сортировка по датам (`created_at`, `date`, `log_date`)
- ✅ Эффективная фильтрация по статусам и ролям

### Масштабируемость
- ✅ Приложение останется быстрым при росте базы данных
- ✅ Оптимизированы JOIN операции через foreign keys
- ✅ Эффективная очистка истекших данных (sessions)

### Типичные запросы, которые стали быстрее
1. Получение всех снимков еды пользователя за период
2. Подсчет непрочитанных уведомлений
3. Получение истории веса пользователя
4. Фильтрация пользователей по роли в админке
5. Поиск активных подписок
6. Очистка истекших сессий

## Рекомендации

### Мониторинг
- Используйте `EXPLAIN ANALYZE` для проверки использования индексов
- Мониторьте медленные запросы через PostgreSQL logs

### Обслуживание
- Периодически запускайте `VACUUM ANALYZE` для обновления статистики
- Следите за размером индексов через `pg_indexes`

### Дополнительные индексы (при необходимости)
Если появятся новые частые запросы, рассмотрите добавление:
- Составных индексов (например, `[user_id, created_at]`)
- Частичных индексов (например, только для активных подписок)
- GIN индексы для полнотекстового поиска
